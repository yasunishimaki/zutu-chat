# 開発ステップリスト

頭痛博士アプリを公開するまでのステップです。上から順に進めますが、「待ち」の項目は並行して進めます。

## ステップ0：準備（今動けること）

- [x] 概要・構成ドキュメントの確定（B案：動画単位の紐付け）
- [x] スプレッドシート列仕様の確定・共有用ドキュメント化
- [ ] サンプルデータの入手（動画シート2〜3行＋先生一覧）※依頼中
- [ ] OpenAI APIキーの用意（File Search・Responses・STT用）
- [ ] Cloudflareアカウントの確認（Pages・Worker・D1・R2・KVが使える状態か）

**待ちの間に並行で進めること**

- [ ] 医師への氏名・所属の掲載許諾確認（動画出演の了承とは別件。公開前までに完了）
- [ ] 動画のYouTube限定公開アップロード（IDはあとからシートに追記）

## ステップ1：RAGコアの技術検証（最重要・最初にやる）

サンプルデータが届いたら着手。ここが成立しなければ構成を見直すため、画面より先に検証する。

- [ ] サンプル2〜3本をFile Search登録形式（1動画=1ファイル）に変換
- [ ] Vector Storeを作成し、ファイルを登録
- [ ] 検証スクリプトで「質問 → 引用付き回答 → 動画IDの取得」が成立するか確認
- [ ] クエリ拡張（普段の言葉→専門用語の言い換え）の効果を確認
- [ ] 検索スコアの分布を見て「該当なし」閾値の当たりを付ける

## ステップ2：データパイプライン（変換・登録スキルの作成）

- [ ] シート読み込み → 登録パッケージ生成（File Search用ファイル／カテゴリー候補／一般向けキーワード／説明文／質問候補）
- [ ] 生成結果をシートに書き戻し、人が確認して確定する流れの確立
- [ ] R2・File Search・D1への一括登録処理（必ずこの経路を通す）
- [ ] File SearchとD1の整合性チェック処理

## ステップ3：Worker API

- [ ] 安全判定（緊急・診断系の質問 → 医師確認済みの固定文に分岐)
- [ ] KVキャッシュ（定番質問はAPIを呼ばず即返す）
- [ ] 回答パイプライン（クエリ拡張 → File Search検索 → 閾値判定 → Responses APIで回答生成 → 引用をD1と突き合わせて動画カード化）
- [ ] 音声認識（STT）の中継
- [ ] 悪用・コスト対策（レート制限・Turnstile・トークン上限・利用量アラート）
- [ ] APIキーをCloudflare Secretへ

## ステップ4：フロントエンド

- [ ] ホーム（博士キャラ・質問例・動画への入口）
- [ ] チャット（ストリーミング表示・音声/テキスト切替・根拠動画カード・関連質問）
- [ ] 動画一覧（検索・カテゴリー・動画カード：タイトル/説明/監修医師）
- [ ] 動画視聴（YouTube埋め込み・文字起こし表示）
- [ ] 監修医師紹介ページ（氏名・所属・専門科、動画制作時点の情報と明示）
- [ ] 管理画面（Cloudflare Accessで認証。登録・公開/非公開切替・整合性チェック）

## ステップ5：データ整備と公開準備

- [ ] 全60本をスキルで一括登録（すべて非公開で登録）
- [ ] YouTube動画IDをシートに追記 → D1へ反映
- [ ] 一般向け動画の選定（公開対象○の付与）
- [ ] 定番質問20〜30問と回答を確定 → KVへ登録
- [ ] 医師レビュー（緊急時固定文の監修・テスト質問への回答確認）
- [ ] 文言ページの整備（免責事項／利用規約／プライバシーポリシー：質問内容がOpenAIに送信されることを明示）
- [ ] 掲載許諾の完了確認（先生一覧シートの許諾列がすべて○）

## ステップ6：公開

- [ ] URL限定の試験公開（応答品質・コスト・速度を監視）
- [ ] 問題なければ一般公開
- [ ] 公開後の運用（利用量アラート・整合性チェックの定期実行）

## 判断済みの方針（迷ったらここに戻る）

- RAGは動画単位の紐付け（B案）。該当場面からの再生は将来拡張
- D1が唯一の台帳。操作は必ず管理画面（またはスキル）経由。削除せず非公開フラグ
- チャット回答に医師名は出さない。動画カードと紹介ページで医師を明示
- 音声認識は最初からOpenAI STT（費用は1質問0.01〜0.02円程度で問題なし）
